# 天達大班報到系統 - 詳細功能說明與邏輯分析

透過分析 `Demo.html` 網頁原始碼，本文件彙整了該系統的介面設計、功能模組、底層邏輯與技術特徵。此系統主要用於管理「天達大班」的班員名單與報到作業，且應是藉由 **Glide Apps** (或其他類似之 Low-code / No-code 平台) 生成的單頁應用程式 (SPA) 與漸進式網頁應用 (PWA)。

## 1. 系統概述 (System Overview)
- **應用名稱**：天達大班報到系統 (Check-in System - Tianda Class)
- **前端類型**：單頁應用程式 (SPA - Single Page Application)，並具備漸進式網頁應用 (PWA) 特性。
- **目標用途**：供系統管理員/班級幹部管理班員資料、追蹤出缺席狀況，並提供現場報到功能（包含手動報到與掃描 QR Code 報到）。
- **身分驗證**：系統底層串接 Firebase Authentication，目前操作者身分為 `Terry G` (`dreamgen@gmail.com`)。

## 2. 核心功能模組：詳細說明 (Detailed Functional Modules)

根據側邊選單 (Sidebar) 與主畫面呈現的元素，系統功能可劃分為以下四大類，各功能的詳細預期行為如下：

### 2.1 學員專屬主頁與入口 (Student Portal & Dashboard)

*   **主頁_已註冊學員 (Home Page - Registered Member)**
    *   **顯示位置**：左側導覽選單的「主頁」。這是一般登入學員（例如：`h.dreamgen@1-kuan-tao.org.tw`）進入系統後的預設儀表板。
    *   **操作邏輯 (依據「主頁_已註冊學員.html」快照解析)**：
        *   **個人身份看板**：頁面上方會顯示專屬的橫幅或看板，載明應用名稱「報到系統-天達大班」。
        *   **個人基本資料**：清楚呈現登入者的頭像 (如：簡)、姓名 (如：簡嘉伸) 以及專屬編號/學號 (如：H311310080)。
        *   **快速操作按鈕**：提供針對學員個人的快捷功能按鈕群：
            *   **「請假」**：允許學員直接線上提出請假申請。
            *   **「個人資訊」**：點擊可進入編輯或檢視更詳細的個人聯絡資料。
    *   **應用場景**：提供已註冊班員專屬的單一入口。學員可在此確認自己的學號與身分，並能快速進行線上請假或維護個人資料，提升自助服務便利性。

### 2.2 報到相關模組 (Check-in Modules)

*   **掃描名牌 QRCODE (QR Code Scanning)**
    *   **顯示位置**：主畫面中段的明顯按鈕 (`<button aria-label="掃描名牌QRCODE">`)。
    *   **操作邏輯**：點擊後會呼叫行動裝置（手機/平板）的相機權限開啟掃描器。使用者對準班員的專屬 QR Code 或條碼名牌掃描後，系統會在背景比對資料庫，若吻合則將該名班員的出席狀態標記為「已報到」，並可能彈出成功提示 (Toast)。
    *   **應用場景**：活動現場快速消化報到人流，班員自行感應或由招待人員協助掃描。
    *   **附屬說明文字**：「您如果已取得名牌或QRCODE, 可掃描後對應班員身份」。

*   **簡易報到 (Simple Check-in)**
    *   **顯示位置**：左側導覽選單。
    *   **操作邏輯 (依據「簡易報到.html」快照解析)**：
        *   **多維度篩選與搜尋**：
            *   **依姓氏搜尋 (使用姓氏查詢)**：快速過濾特定姓名的班員。
            *   **依單位/區別 (區域)**：如三重區、內湖區、基隆區、新莊區、松山區、瑞芳區等進行篩選。
            *   **狀態過濾**：可切換檢視「已出席」或「未出席」名單。
        *   **名單呈現與分頁**：列表分頁顯示符合條件的成員（如：丁國恒、丁義國等），底部有 Previous/Next 及頁碼導覽。
        *   **報到狀態切換**：清單中提供「未到」/「送出報到」等狀態切換開關，讓管理員可以極速完成手動核實。
        *   **功能擴充設定**：
            *   **更換班別**：允許現場臨時調整學員的上課班別。
            *   **大字版**：考量到接待志工可能有長輩，提供字體放大的友善介面。
            *   **備註與關係人**：提供「請輸入關係人或是到班因由」的輸入框，註記該次報到的特殊狀況。
            *   **臨時報到選項**：若為新面孔或未在冊人員，支援「若無現存的使用者，請輸入完整姓名後，按臨時報到」功能，直接建立臨時紀錄。
        *   **即時統計**：畫面上方或側邊顯示「簡易報到人數(含臨時報到)」的即時總計數據。
    *   **應用場景**：提供接待處人員極大彈性的手動報到手段，應對忘記帶名牌、無資料的新人、以及長輩眼力不佳需使用大字版的現場實況。

### 2.3 班員與資料管理模組 (Member & Data Management)

*   **班員資料 (Class Member Data)**
    *   **顯示位置**：左側導覽選單。
    *   **操作邏輯 (依據「班員資料.html」與「班員資料_詳細資料.html」快照解析)**：該模組分為「列表檢視」與「詳細資料」雙層架構。
        *   **總覽列表檢視 (List View)**：
            *   **清單呈現**：所有班員以「卡片 (Card)」的垂直集合形式列出。支援分頁控制 (如：1 到 43 頁)。
            *   **快速過濾與搜尋**：頂部具備文本搜尋框 (Search) 與進階篩選 (Filter) 按鈕。
            *   **卡片資訊**：每張卡片展示班員姓名 (如：丁國恒)、學號/編號 (如：H111310001)、以及加入的個別社團與志工服務標籤 (如：身心舒活, 志工服務, 烹飪社)。
            *   **特殊名單**：提供「臨時報到名單」的快速入口或切換按鈕。
        *   **詳細資料檢視 (Detailed View)**：點擊特定班員卡片可進入個人專屬頁面。
            *   **基本檔案**：頂部顯示個人大頭照 (支援「新增照片」操作)、姓名與學號。
            *   **群組歸屬資訊**：列出該員的「所屬單位 (如：瑞芳區)」、「班級 (如：明理班,明德班)」與基本屬性「乾坤 (性別，如：乾)」。
            *   **歷史出席紀錄 (Data Grid)**：頁面下方以完整的數據表格 (Data Grid Canvas) 呈現該員的歷年出席狀態。欄位包含：「週次、月份、日期、班別、狀態」(例如：第5週，七月，7/20/2025，青推日-社團)。此表格讓輔導幹部能詳細追蹤個人的穩定度與出席歷程。
    *   **應用場景**：提供單一班員全方位的檔案管理與歷程檢視，方便小組長/輔導人員針對特定對象進行關懷、追蹤個別出席率並完善資料。

*   **班員資料_分班檢視 (Class Member Data - Class View)**
    *   **顯示位置**：左側導覽選單。
    *   **操作邏輯**：資料來源與「班員資料」相同，但採用了「分組 (Group by)」或「標籤式過濾 (Tab filter)」的檢視邏輯。畫面可能會先顯示各個班級或小組的名稱，點入後才顯示該組的成員。
    *   **應用場景**：方便各小組長/輔導幹部只看自己負責的組員，進行分組點名或關懷。

*   **快速查詢 (Quick Search)**
    *   **顯示位置**：左側導覽選單，點擊後進入專屬的「快速查詢」頁面。
    *   **操作邏輯 (依據「快速查詢.html」快照解析)**：
        *   **複合式篩選條件**：提供多種下拉選單與輸入框進行精準搜尋，包含：
            *   **查詢日期**：可選擇特定日期（如：2026/3/1）。
            *   **查詢班別**：下拉選擇特定班別（如：第一週日）。
            *   **查詢單位**與**查詢班級**：進階下拉篩選。
            *   **查詢姓名**：提供文字輸入框直接搜尋特定對象。
            *   **查詢狀態**：提供多選狀態過濾，包含「輔導出席、輔導線上、學員出席、學員線上、缺席、遲到」。
        *   **姓名批次查詢**：具備「姓名批次查詢」快捷按鈕，預期可執行進階的批次處理或查詢邏輯。
        *   **快速文字過濾**：在結果列表上方，額外提供一個含 Search 與 Clear (清除) 按鈕的搜尋框，對當下結果進行二次過濾。
        *   **結果呈現**：搜尋結果會以「卡片清單 (Card Collection)」的形式展現。每張卡片標示班員姓名（如：尹順隆）、性別/身分註記 (如：乾、坤) 以及出席狀態圖示 (如：○)。
        *   **分頁控制**：當符合條件的人數眾多時，底部提供分頁按鈕 (Previous, Next, 以及頁碼 1~13...) 供使用者翻頁瀏覽。
    *   **應用場景**：當班級人數眾多，需針對特定日期、狀態（如：找出所有缺課學員）、特定單位進行名單盤點與追蹤時，管理員可透過此功能快速篩選出目標班員群體，確認報到狀態或進行後續跟進。

### 2.3 統計與報表模組 (Statistics & Reporting)

*   **出席統計 (Attendance Statistics)**
    *   **顯示位置**：左側導覽選單。
    *   **操作邏輯 (依據「出席統計.html」快照解析)**：
        *   **條件篩選**：頂部提供下拉選單進行資料過濾：
            *   **查詢日期**：僅顯示已進行之日期（例如：2026/3/1）。
            *   **查詢班別**：可選擇特定班別（例如：第一週日）。
        *   **總體數據儀表板**：以醒目的數字卡片呈現當日總體出席概況，包含：實到總人數、乾 (男性人數)、坤 (女性人數)、以及臨時報到人數。
        *   **各區/各班明細表 (簡易報到-各區人數)**：
            *   提供「顯示比較」的快速切換開關 (Toggle Switch)。
            *   以表格呈現各區 (如：基隆區、瑞芳區、內湖區、松山區、三重區、新莊區) 以及各個別編制 (如：明理班、新民班、小天使班) 的實際報到人數。
        *   **數據視覺化長條圖**：
            *   頁面底部提供互動式長條圖 (Bar Chart)，視覺化展示各區別的人數分佈差異，圖例標示支援所有主要分區，讓管理者一目了然各區出席狀況。
    *   **應用場景**：供主事者或班級幹部在大規模活動後，透過儀表板與視覺化長條圖，快速且精確地掌握各區、各班的出席與性別比例結構，便於後續關懷追蹤。

### 2.4 系統與排程管理模組 (System & Schedule Management)

*   **班程資料 (Class Schedule Data)**
    *   **顯示位置**：左側導覽選單。
    *   **操作邏輯**：預期為一個行事曆或課程列表介面。每一筆紀錄代表一次上課或活動（如：2026-03-12 專題演講）。點擊特定班程，可能可以設定該次的報到開放時間，或是查看該單次活動的專屬報到統計。
    *   **應用場景**：管理全學期的課程表設定，切換當次報到所對應的「活動場次」。

*   **工具設定 (Tool Settings)**
    *   **顯示位置**：左側導覽選單。
    *   **操作邏輯 (依據「工具設定.html」快照解析)**：此模組主要用於針對不同平台與情境，產生報到系統或掃描器的專屬設定與入口連結。
        *   **參數設定區**：提供管理員指定當次活動的參數，包含：
            *   **選擇日期**與**選擇班別** (例如：2026/3/15，第三週日)。
            *   **模式切換**：可選擇「實體」或「線上」。
            *   **建置包名稱**：可變更對應配置的名稱 (例如：天達大班-實體3)。
            *   **相容舊QRCODE**：提供相容性開關，以繼續支援舊版格式的 QR Code 或名牌。
        *   **APP設定與派發**：此區域標題為「APP自動設定QRCODE」，根據上方設定的參數，系統會提供不同平台的快速配置：
            *   **自動完成設定** (涵蓋 Android 與 Web 平台)。
            *   **自動報到系統APP** 專屬入口或設定參數 (分為 Android 與 Web 版本)。
            *   **掃碼APP** 專屬入口或設定參數 (支援 iOS 與 Android 雙系統)。
        *   **分享功能**：畫面底部具備「分享本系統」按鈕，方便將設定好的應用程式或系統入口快速傳遞給現場工作人員。
    *   **應用場景**：在報到活動開始前，資訊人員或管理員在此預先設定好當日的日期與班別，並勾選所需的相容性選項，隨即產生各個輔助 APP (自動報到/掃碼工具) 的配置檔或專屬 QR Code，讓現場執行點名操作的幹部能快速就緒並連接同一個雲端作業場次。
