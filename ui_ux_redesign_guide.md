# 天達大班報到系統 - UI/UX 重構設計說明書

> **使用說明**：本文件為依據功能快照分析所撰寫的 UI/UX 重構規格書，供後續 AI 或開發人員重新設計系統介面時作為完整依據。所有功能模組均已涵蓋。

---

## 1. 系統背景與重構目標

- **現況**：系統由 Glide Apps（Low-code/No-code 平台）建構，為 SPA + PWA 應用。介面受限於平台框架，靈活度與視覺質感有改善空間。
- **身份驗證**：Firebase Authentication（角色：管理員 `dreamgen@gmail.com`、一般學員 `h.dreamgen@1-kuan-tao.org.tw`）。
- **重構目標**：以原生 Web App（如 Next.js / Vite + Tailwind CSS / Vanilla CSS）重新實作，達成：
  - 清晰的角色分流（學員端 vs. 管理端）
  - 更佳的行動裝置體驗（Mobile First）
  - 現代化、高視覺質感的 UI

---

## 2. 目標使用者與角色分流

| 角色 | 主要任務 | 設計重點 |
|------|----------|----------|
| **一般學員 (Student)** | 確認個人狀態、請假、更新個人資料 | 簡潔、直覺、單一焦點 |
| **管理員／幹部 (Admin/Staff)** | QR 掃描報到、手動點名、查詢、統計、系統設定 | 高效率、批次操作、資訊密度高 |

> **重構原則**：登入後依據身份（Firebase 角色）自動切換至對應的佈局與導覽結構，而非顯示所有功能。

---

## 3. 導覽架構設計 (Navigation Architecture)

### 3.1 行動端（Mobile）

採用 **底部導覽列（Bottom Navigation Bar）**，放置最核心的 4 個入口，避免漢堡選單的隱藏感：

| Tab | 圖示 | 對應功能 |
|-----|------|----------|
| 主頁 | 🏠 | 學員個人儀表板 / 管理員快速操作 |
| 掃描 | 📷 | 掃描 QR Code 報到（僅 Admin 見） |
| 查詢 | 🔍 | 快速查詢 + 班員資料 |
| 統計 | 📊 | 出席統計儀表板 |
| 更多 | ⋯ | 班員資料分班檢視 / 班程資料 / 工具設定 |

### 3.2 桌面端（Desktop / Tablet）

採用**左側固定式側邊欄（Sidebar）**，可折疊。側邊欄依角色顯示不同選單項目：

**管理員側邊欄群組：**
1. 主頁（即時檢視）
2. --- 報到 ---
3. 掃描名牌 QR Code
4. 簡易報到（手動點名）
5. --- 資料 ---
6. 班員資料
7. 班員資料_分班檢視
8. 快速查詢
9. --- 統計 ---
10. 出席統計
11. --- 管理 ---
12. 班程資料
13. 工具設定
14. 權限表 / 使用者管理

### 4.1 學員主頁（Student Dashboard）

**對應原功能**：`主頁_已註冊學員`

**頁面目標**：讓學員一進入系統就看到個人狀態與快捷操作，無需跳轉多頁。

**版面設計**：
```
┌──────────────────────────────────┐
│  [橫幅] 報到系統 - 天達大班        │
│         版本描述 / 今日日期         │
├──────────────────────────────────┤
│  ┌────────────────────────────┐  │
│  │  [頭像大圖]  簡嘉伸          │  │  ← 數位名牌卡
│  │  學號：H311310080           │  │
│  │  所屬班：明理班              │  │
│  │  今日狀態：[🟢 已報到]       │  │
│  │  ┌─────────────────────┐  │  │
│  │  │   [個人 QR Code]     │  │  │  ← 供幹部掃描
│  │  └─────────────────────┘  │  │
│  └────────────────────────────┘  │
│                                  │
│  ┌──────────┐   ┌──────────────┐ │
│  │  📋 請假  │   │  👤 個人資訊  │ │  ← 主要操作按鈕
│  └──────────┘   └──────────────┘ │
└──────────────────────────────────┘
```

**設計要求**：
- 「今日狀態」：以語義色顯示（🟢 已報到／🔴 未報到／⚪ 已請假）。
- 個人 QR Code 顯示學員編號，供幹部端掃描機使用，不需回主系統查。
- 「請假」、「個人資訊」設計為全寬大按鈕（48px 高以上），易於拇指觸控。

### 4.2 掃描名牌 QR Code（QR Code Scanner）

**對應原功能**：主頁掃描按鈕

**版面設計**：
```
┌──────────────────────────────────┐
│  [← 返回]     掃描報到           │
├──────────────────────────────────┤
│  ┌────────────────────────────┐  │
│  │                            │  │
│  │   [相機即時預覽窗格]          │  │
│  │       ┌────────────┐       │  │
│  │       │  對焦框     │       │  │
│  │       └────────────┘       │  │
│  │                            │  │
│  └────────────────────────────┘  │
│                                  │
│  [🔦 手電筒] .............. [手動] │
│                                  │
│  [成功訊息區（Toast／彈出卡片）]   │
└──────────────────────────────────┘
```

**設計要求**：
- 相機預覽佔據至少 60% 螢幕高度，讓對焦框清晰可見。
- 掃描成功後：全畫面閃爍綠色 + 班員姓名大字彈出（2 秒後自動關閉），接著立即可繼續連續掃描。
- 掃描失敗（查無此人）：紅色閃爍 + 「查無此編號」提示。
- 「手動」按鈕可切換為輸入編號手動報到。
- 使用 **`zxing-js/browser`** 函式庫實作（`@zxing/browser` npm 套件）。

### 4.3 簡易報到（Manual Check-in List）

**對應原功能**：`簡易報到`

**頁面目標**：讓接待管理員能極速手動處理無名牌的班員，甚至現場新建臨時紀錄。

**版面設計**：
```text
┌──────────────────────────────────┐
│  簡易報到 - 總計人數: 185 人         │
│  [🔍 依姓氏搜尋...] [顯示: 未報到▼]  │
│  [區域▼]  [班別▼]  [🔠 大字版]    │
│  ──────────────────────────────  │
│  [一鍵全選]                     │
│  ──────────────────────────────  │
│  ○ 丁國恒  H111310001  [乾]      │
│    └ ✏️ 備註/關係人...             │
│  ✓ 丁義國  H211310005  [乾]  ← 開關
│  ○ 于琴玲  H311320125  [坤]      │
│    [🔄 更換班別]                  │
│  ...                             │
│                                  │
│  [← 上一頁]  1/43  [下一頁 →]    │
│  ──────────────────────────────  │
│  [➕ 找不到人？輸入全名新增臨時報到]  │
└──────────────────────────────────┘
```

**設計要求**：
- **複合過濾器**：頂部需提供「姓氏搜尋」、「未到/已到 狀態」、「區域」、「班別」等快速過濾選項。
- **操作元件**：每列右側為大型 Toggle/Checkbox（44px 以上觸控面積），方便快速標記「未到/已到」。
- **視力友善**：提供「大字版」切換開關，讓高齡招待志工能輕鬆操作。
- **行內擴充功能**：
  - 「更換班別」按鈕可臨時為該班員更改上課班級。
  - 「備註輸入框」可選填關係人或是到班因由。
- **臨時報到**：面底部或固定位置提供明顯的「臨時報到」功能，直接輸入全名新增非在冊人員。

- 「姓名批次查詢」作為 Drawer 內的獨立按鈕，點開後提供多行文字框供批次輸入。
- 支援多選模式（長按開啟），批次標記出席狀態。

## 5. UI 視覺風格規格 (Visual Design System)

### 5.1 色彩計畫（Color Palette）

| 用途 | 顏色名稱 | Hex 值 |
|------|----------|--------|
| 主色 Primary | 海水藍 | `#2F6783` |
| 主色深 Primary Dark | 深藍 | `#1D4D63` |
| 主色淡 Primary Light | 淡藍 | `#E0EDF5` |
| 成功 Success | 翠綠 | `#10B97A` |
| 警告 Warning | 琥珀黃 | `#FBBF24` |
| 錯誤/缺席 Danger | 珊瑚紅 | `#F0554C` |
| 請假 Neutral | 雲灰 | `#8F8F8F` |
| 背景 Background | 暖白 | `#F5F7FA` |
| 卡片 Card | 純白 | `#FFFFFF` |
| 文字主色 Text | 深碳灰 | `#1A202C` |
| 文字次要 Subtext | 霧灰 | `#6B7280` |

### 5.2 字體排版（Typography）

- **字體**：`Noto Sans TC`（中文） + `Inter`（英數字），透過 Google Fonts 載入。
- **字級層次**：
  - 標題 H1：24px / 700
  - 頁面子標題 H2：18px / 600
  - 卡片標題：16px / 600
  - 內文 Body：15px / 400
  - 輔助說明 Caption：13px / 400，色：Subtext
- **行高**：1.6（中文閱讀）

### 5.3 圓角與陰影（Border Radius & Shadow）

- 卡片：`border-radius: 12px`
- 按鈕：`border-radius: 8px`
- 輸入框：`border-radius: 8px`
- 卡片陰影：`box-shadow: 0 2px 8px rgba(0,0,0,0.08)`

### 5.4 夜間模式（Dark Mode）

支援 `prefers-color-scheme: dark` 自動切換，定義 CSS 變數：

```css
:root {
  --color-bg: #F5F7FA;
  --color-card: #FFFFFF;
  --color-text: #1A202C;
  --color-primary: #2F6783;
}

@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #0F1923;
    --color-card: #1C2B3A;
    --color-text: #E2E8F0;
    --color-primary: #5BA8C8;
  }
}
```

## 6. 通用 UX 組件規格 (Reusable Components)

| 組件 | 規格說明 |
|------|----------|
| **Toast 通知** | 右上角或底部彈出，自動 2~3 秒消失，支援成功/警告/錯誤三種語義 |
| **Bottom Sheet** | 行動端通用，半透明背景遮蔽，可拖動，適用篩選器、QR Code 顯示、確認框 |
| **Status Badge** | 出席 🟢 / 缺席 🔴 / 請假 ⚪ / 線上 🔵，帶圓點的文字標籤 |
| **Pill Badge** | 圓角標籤元件，用於社團、班別等標籤顯示，最多顯示 3 個，超出顯示 +N |
| **Loading Skeleton** | 卡片骨架屏，提升載入中的視覺品質，優於 spinner |
| **Empty State** | 查詢無結果時顯示插圖 + 引導文字 |
| **KPI Card** | 大數字 + Count Up 動畫 + 下方標籤說明 |
